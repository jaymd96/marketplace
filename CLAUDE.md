# CLAUDE.md

Personal Claude Code plugin marketplace managed by the `guild` CLI.

## Quick Reference

```bash
guild list                          # Show all registered plugins
guild build                         # Regenerate marketplace.json + README.md
guild validate                      # Check marketplace structure is valid
guild add <name>                    # Scaffold a new plugin in plugins/<name>/
guild add <name> --description "…"  # Scaffold with description
guild remove <name>                 # Remove a plugin
guild version <name> <version>      # Set a plugin's version
guild setup-plugin [target]         # Install guild's own skills plugin
```

## Adding a New Plugin

1. Create the plugin directory with its components:
   ```
   plugins/<name>/
     plugin.toml          # Required — name, version, description
     CLAUDE.md            # Plugin documentation
     skills/              # Invocable /commands
     agents/              # Subagents (isolated context)
     hooks/               # Event-driven automation (hooks.json)
     scripts/             # Shell scripts for mechanical tasks
     system/              # Harness internals (ENTRYPOINT.md, workflows/)
   ```

2. Register and build:
   ```bash
   guild build            # Scans plugins/, regenerates marketplace.json + README.md
   ```
   Note: `guild add <name>` scaffolds a new empty plugin. If the directory
   already exists (you built it manually), just run `guild build` to register it.

3. Commit and push:
   ```bash
   git add plugins/<name>/ marketplace.json README.md
   git commit -m "Add <name> plugin: <short description>"
   git push origin main
   ```

## Plugin Manifest (plugin.toml)

```toml
[plugin]
name = "my-plugin"
version = "0.1.0"
description = "What this plugin does in one sentence."
```

`guild build` reads every `plugins/*/plugin.toml` to generate `marketplace.json`.

## Structure

```
marketplace/
  guild.toml               # Marketplace metadata (name, owner)
  marketplace.json         # Generated — plugin registry (do not edit manually)
  README.md                # Generated — plugin table (do not edit manually)
  setup.sh                 # Full setup: build + validate + install guild-skills
  .claude-plugin/
    marketplace.json       # Symlink → ../marketplace.json
  plugins/
    codecraft/             # Spec → working code (state machine methodology)
    pytest-testing/        # pytest test writing and running
    python-toolkit/        # Python coding standards + library references
    research-builder/      # Research idea refinement
    spec-builder/          # Conversation → specifications
```

## Key Rules

- **Never edit marketplace.json or README.md manually** — they're generated by `guild build`
- **Always run `guild build` after adding/changing plugins** — keeps the registry in sync
- **Run `guild validate` before pushing** — catches structural issues
- **plugin.toml is the source of truth** — guild reads name, version, description from it
